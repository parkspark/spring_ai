Prompt Engineering Guide

Zero-shot vs Few-shot

프롬프팅 방식에 따른 AI 모델의 응답 품질과 특성 차이를 한눈에 비교하고 시뮬레이션 해보세요.

Zero-shot (제로샷)

모델에게 사전 예시를 제공하지 않고 오직 지시사항만으로 작업을 수행하도록 요청하는 가장 기본적인 방식입니다.

Prompt: 다음 문장의 감성을 분석해줘: "이 영화 진짜 시간 낭비였어."

Few-shot (퓨샷)

원하는 결과물의 예시(1개 이상)를 함께 제공하여 모델이 패턴을 학습하고 의도한 형식에 맞게 출력하도록 유도하는 방식입니다.

Prompt: 문장 감성 분석
예시1: "최고의 경험이었어요!" -> [긍정]
문제: "이 영화 진짜 시간 낭비였어." -> ?

실제 비교 예시 (상품 리뷰 분석)

Zero-shot

입력 (Prompt)

다음 리뷰의 핵심 키워드를 추출해줘.
리뷰: "배송은 하루만에 와서 좋았는데, 포장이 다 뜯어져 있어서 기분이 별로였어요. 맛은 그럭저럭 괜찮네요."

출력 (Output)

이 리뷰에서 추출할 수 있는 핵심 키워드는 다음과 같습니다:
1. 배송 (하루만에 옴)
2. 포장상태 (뜯어져 있음, 불만족)
3. 맛 (보통)
// 설명이 길고 포맷이 일정하지 않음
Few-shot

입력 (Prompt)

리뷰에서 [장점, 단점, 별점] 형식으로 추출해.
리뷰: "가성비 최고! 근데 배터리가 빨리 닳음" -> [장점:가격, 단점:배터리, 별점:4]
리뷰: "배송은 하루만에 와서 좋았는데, 포장이 다 뜯어져 있어서 기분이 별로였어요. 맛은 그럭저럭 괜찮네요." -> ?

출력 (Output)

[장점:배송, 단점:포장, 별점:3]

// 예시에서 지시한 정확한 포맷으로 간결하게 출력됨

장단점 및 특징 비교

비교 항목 Zero-shot Few-shot
정확도 & 일관성 일반적인 작업은 괜찮으나, 특수한 포맷 요구시 실패 확률 높음 예시를 통해 패턴을 학습하므로 정확도와 포맷 일관성이 매우 높음
토큰 소모량 (비용) 예시가 없으므로 토큰 소모가 적고 경제적 예시 데이터만큼 프롬프트가 길어져 비용 증가
처리 속도 (응답 시간) 입력이 짧아 비교적 빠름 입력이 길어 약간 느릴 수 있음
적합한 유즈케이스 번역, 요약, 일반적인 질문답변, 아이디어 브레인스토밍 복잡한 데이터 파싱, 감성 분석, 특정 JSON 포맷 변환
